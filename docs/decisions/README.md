# Design Decisions

Record significant decisions made during the WID 3.0 spec development here — field names
chosen, endpoints added or removed, data type choices, etc.

## Format

Name each file with a date and short slug, e.g. `2026-04-06-employer-id-field-name.md`.

Each file should cover:
- **Decision:** What was decided
- **Context:** Why this came up
- **Options considered:** What alternatives were discussed
- **Rationale:** Why this option was chosen
- **States involved:** Who participated in the discussion
