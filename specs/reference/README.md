# Reference Specifications

This directory holds existing WID-compatible API specifications contributed by states as
reference material for the WID 3.0 working draft.

## Adding a reference spec

1. Name the file `<state>-wid-<version>.yaml`, e.g. `oregon-wid-2.8.yaml`
2. Add an entry for it in `index.html` so it appears in the Swagger UI dropdown
3. Open a pull request
